有意义的命名

规则

名副其实

  1. 起名字是一件很严肃的事情
  2. 一旦发现更好的名称，就替换掉旧的(需要预估替换的成本)
  3. 名称应该已经答复了所有的大问题(如果需要注释补充就不算名副其实)
const d: number; // 消逝的时间，以日记
const elapsedTimeInDays: number;
const theList = [
  { index: 1 }, { index: 2 }, { index: 3 }, { index: 4 }, 
];
function getList(): number[] {
  const list: number[] = [];
  theList.forEach(i => {
    if (i.index === 4) {
      list.push(i);
    }
  })
  return list;
}
class Cell {
  static flagStatus = 4;
  public status: number;
  constructor(status: number) {
    this.status = status;
  }
  isFlagged() {
    // 是否标识的4 可以用一个常量代替
    return true;
  }
}
const gameBoard = [
  new Cell(1), new Cell(2), new Cell(3), new Cell(4), 
];
function getFlaggedCells(): Cell[] {
  const flaggedCells: Cell[] = [];
  gameBoard.forEach(cell => {
    if (cell.isFlagged()) {
      flaggedCells.push(cell);
    }
  })
  return flaggedCells
}

避免误导

  1. 一些通用的关键名词不要用来命名 hp、aix、sco UNIX的专有名词
  2. 避免出现极其相似的单词
    const XYZControllerForEfficientHandLingOfSettings: string;
const XYZControllerForEfficientStorageOfSettings: string;
  3. 不要单独用l O 做名字
    l O 和 1 0 比较像
  4. 做有意义的区分 不要同时用xxx0，xxx1 等来命名多个变量
    不要同时用Product ProductInfo ProductData 和使用a an the是废话
  5. 使用读得出来的名称 使用日常单词
    // 反例
const XLQY;
const FCNV;
const LTQManager;
    // 一些通用的缩写
const conf; // config
  6. 使用可搜索的名称 避免使用一些单词中出现出现频率极高的一些短字母组合
    const ab = 'xxx';
    ps: 6、7 两点凸显了英语的重要性
  7. 避免使用编码 
    const 0x649 = 'xxxx'; // 列举了一个16进制的编码
  8. 避免使用无意义的前/后缀
    虽然是这样说，但是还是有一些很常见的前缀命名法
    const m_info = 'xxx';
interface IProps {} // 前缀的接口
const $timeChange = new Observable(); // $前缀代表Observable
class ShapeFactoryImp {} // 尾缀的抽象类
  9. 每一个概念对应一个词，一个语意用一个词写完
    不用同一个意思出现多个不同的变量 
    const Controller; // 管理员 管理者
const Manager; // 管理员 管理者
const ControllerManger;
  10. 双关语，避免用同一术语代表不同的概念 
    // 同一个类中有两个不同的插入
class A {
  add() {}
  insert() {}
  append() {}
}
  11. 使用计算机相关的名称
  12. 为变量增加有意义的语境 
    const firstName;
const lastName;
const street;
    const AddrfirstName;
const AddrLastName;
const AddrStreet;

那些需要命名

  • 变量

    函数 函数或方法应该是动词或动词短语

function postPayment() {}
function deletePage() {}
function saveXXX() {}
  • 参数

    类 类名和对象应该是名词或名词短语

class Customer {}
class Account {}
class Manager {}
class XXXInfo {}
  • 包
  • 文件
    xxx.component.ts
xxx.pipe.ts
xxx.service.ts
  • 目录 父级目录实际上提供了一个前置的语境
    payXXX
  payOrder
  payList

  // 对应具体文件中的类名或其他的名字是需要加上Pay的
  class PayOrder {}
  class PayList {}
